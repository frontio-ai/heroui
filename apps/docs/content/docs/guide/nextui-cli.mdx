# NextUI CLI

The goal of NextUI CLI is to help you start your NextUI project quickly.

It provides a set of commands to init a new NextUI project, add/remove/upgrade components, Check your project health and more.

## Installation

Requirements:

- [Node.js 18.17.x+](https://nodejs.org/en/) or later

<Steps>

### Global Installation

Run one of the following commands in your terminal to globally install the `NextUI CLI`:

<PackageManagers
  commands={{
    npm: "npm install nextui-cli -g",
    yarn: "yarn add nextui-cli -g",
    pnpm: "pnpm add nextui-cli -g",
    bun: "bun add nextui-cli -g"
  }}
/>

### Without Installation

You can also run the `NextUI CLI` without installing it globally by using `npx`:

```bash
npx nextui-cli@latest
```
</Steps>

## Quick Start

After installing the `NextUI CLI`, run the following command to see the available commands:

```codeBlock bash
nextui
```

And then you can see the `NextUI CLI` help output like this:

```codeBlock bash
NextUI CLI <version>

A CLI tool that unlocks seamless NextUI integration

Usage: nextui [command]

Options:
  -v, --version                      Output the current version
  -h, --help                         Display help for command

Commands:
  init [options] [projectName]       Initialize a new NextUI project
  add [options] [components...]      Add NextUI components to your project
  upgrade [options] [components...]  Upgrade the NextUI components to the latest version
  remove [options] [components...]   Remove NextUI components from your project
  list [options]                     List all the components status, description, version, etc
  env [options]                      Display debug information about the local environment
  doctor [options]                   Check whether exist problem in user project
  help [command]                     display help for command
```

### Init A New Project

`NextUI CLI` Provides a command to init a new NextUI project, you can run the following command to create a new project:

```codeBlock bash
nextui init [my-nextui-app-name]
```

Run the following command `nextui init -h` to See the all available options for the `init` command:

```codeBlock bash
Usage: nextui init [options] [projectName]

Initialize a new NextUI project

Arguments:
  projectName             The name of the new project

Options:
  -t --template [string]  The template to use for the new project e.g. app, pages
  -h, --help              Display help for command
```

### Add Components

You can add NextUI components to your project by using the `add` command:

```codeBlock bash
nextui add [component1] [component2] [component3] ...
```

Run the following command `nextui add -h` to See the all available options for the `add` command:

```codeBlock bash
Usage: nextui add [options] [components...]

Add NextUI components to your project

Arguments:
  components                   The name of the NextUI components to add

Options:
  -a --all [boolean]           Add all the NextUI components (default: false)
  -p --packagePath [string]    The path to the package.json file
  -tw --tailwindPath [string]  The path to the tailwind.config file file
  -app --appPath [string]      The path to the App.tsx file
  --prettier [boolean]         Add prettier format in the add content which required installed prettier (default: false)
  --addApp [boolean]           Add App.tsx file content which required provider (default: false)
  -h, --help                   Display help for command
```

### Upgrade Components

You can upgrade NextUI components to the latest version by using the `upgrade` command:


Run the following command `nextui upgrade -h` to See the all available options for the `upgrade` command:

```codeBlock bash
Usage: nextui upgrade [options] [components...]
```

### List Components

You can list all the components status, description, version, etc by using the `list` command:

```codeBlock bash

```

Run the following command `nextui list -h` to See the all available options for the `list` command:

```codeBlock bash

List all the components status, description, version, etc

  -p --packagePath [string]  The path to the package.json file
  -c --current               List the current installed components
  -h, --help                 Display help for command
```

### Check NextUI Project Health

You can check whether exist problem in your project by using the `doctor` command:

```codeBlock bash
nextui doctor
```

Run the following command `nextui doctor -h` to See the all available options for the `doctor` command:

```codeBlock bash
Usage: nextui doctor [options]

Check whether exist problem in user project

Options:
  -p --packagePath [string]      The path to the package.json file
  -tw --tailwindPath [string]    The path to the tailwind.config file file
  -app --appPath [string]        The path to the App.tsx file
  -ca --checkApp [boolean]       Open check App (default: true)
  -ct --checkTailwind [boolean]  Open check tailwind.config file (default: true)
  -cp --checkPnpm [boolean]      Open check Pnpm (default: true)
  -h, --help                     Display help for command
```
